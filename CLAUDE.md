# SkillBridge 2.0

## Coding style — ALWAYS ON

All code written, edited, refactored, or reviewed in this project follows the
**`plain-readable-code`** skill (`.claude/skills/plain-readable-code/SKILL.md`).

This is the default style for the whole project — not opt-in. Apply it to every
coding task (backend, frontend, scripts, fixes) without being asked. In short:
clear over clever, plain language features first, functions that earn their
existence, boring well-maintained libraries over hand-rolled complexity,
top-down structure, comments explain *why* not *what*.
